MYSQL

MySQL es un sistema de administración de bases de datos relacionales. Es un software de código abierto desarrollado por Oracle. Se considera como la base de datos de código abierto más utilizada en el mundo.

Numéricos:

INT: Almacena números enteros sin decimales.
DECIMAL: Almacena números decimales con una precisión específica.
FLOAT: Almacena números de coma flotante de precisión simple.
REAL: Almacena números de coma flotante de precisión doble.
NUMERIC: Almacena números decimales con una escala y precisión específicas.

precio DECIMAL(10,2): Almacena el precio de un producto con dos decimales.
saldo FLOAT: Almacena el saldo de una cuenta bancaria como un número de coma flotante.

Caracteres:

CHAR(n): Almacena una cadena de caracteres de longitud fija n.
VARCHAR(n): Almacena una cadena de caracteres de longitud variable hasta n caracteres.
TEXT: Almacena grandes cantidades de texto.
NCHAR(n): Almacena una cadena de caracteres Unicode de longitud fija n.
NVARCHAR(n): Almacena una cadena de caracteres Unicode de longitud variable hasta n caracteres.
Ejemplos: nombre VARCHAR(255): Almacena el nombre de una persona con una longitud máxima de 255 caracteres.
direccion TEXT: Almacena la dirección de una persona como un texto largo.
descripcion NVARCHAR(1000): Almacena la descripción de un producto con caracteres Unicode.

Fechas:

DATE: Almacena fechas sin hora.
TIME: Almacena horas sin fecha.
DATETIME: Almacena fechas y horas juntas.
TIMESTAMP: Almacena fechas y horas con precisión de milisegundos.
Ejemplos:
fecha_nacimiento DATE: Almacena la fecha de nacimiento de una persona.
hora_entrada TIME: Almacena la hora de entrada de un empleado.
fecha_registro DATETIME: Almacena la fecha y hora de registro de un usuario.

Modificadores:

NOT NULL: Indica que el campo no puede ser nulo.
UNIQUE: Indica que cada valor en el campo debe ser único.
DEFAULT: Especifica un valor predeterminado para el campo.
PRIMARY KEY: Indica que el campo es la clave principal de la tabla.
FOREIGN KEY: Indica que el campo es una clave foránea que referencia otra tabla.
Ejemplos:
id INT NOT NULL PRIMARY KEY: El campo id es un número entero que no puede ser nulo y es la clave principal de la tabla.
nombre VARCHAR(255) UNIQUE: El campo nombre no puede contener valores duplicados.
correo_electronico VARCHAR(255) DEFAULT 'ejemplo@correo.com': El campo correo_electronico tiene un valor predeterminado de ejemplo@correo.com si no se especifica otro.

Listas:

En SQL, no existe un tipo de dato específico para las listas. Sin embargo, hay varias formas de almacenar listas de datos, como:
Usando una tabla separada: Cada elemento de la lista se almacena como una fila en una tabla separada.
Usando un tipo de dato de cadena: La lista se almacena como una cadena de caracteres separada por comas u otro delimitador.
La mejor manera de almacenar una lista en SQL dependerá de las necesidades específicas de la aplicación.
